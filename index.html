WINDOWS AZURE?

Azure adalah sebuah Platform cloud dari Microsoft yang menyediakan berbagai macam layanan. Seperti: Komputasi, Web+Mobile, Data+Storage, Internet of Things, dan masih banyak lagi.

Untuk saat ini pusat data dari Microsoft Azure tersebar di wilayah Amerika Serikat, Eropa, dan Asia.

Selain itu Windows Azure juga memiliki CDN (Content Delivery Network) sebuah sistem yang memungkinkan Windows Azure untuk mengirim/menyampaikan konten dengan bandwidth tinggi. Pada saat ini CDN tersebut berada di 24 situs yang berbeda di Amerika Serikat, Eropa, Australia, dan Amerika Selatan.

Dengan Windows Azure Anda dapat membangun dan mengelola solusi hampir untuk semua tujuan. Microsoft Azure juga menyediakan platform untuk membangun sebuah aplikasi yang memanfaatkan Cloud untuk memenuhi kebutuhan sebuah bisnis.

Pada umumnya setiap bisnis mempertimbangkan untuk memindahkan aplikasi mereka ke Cloud, untuk salah satu dari tiga alasan seperti kecepatan, Skala, dan Ekonomi.

Tutorial Menggunakan Azure Web+Apps Services

Pada tutorial kali ini saya akan memberikan langkah-langkah menggunakan layanan Azure WEB+APP, beberapa hal yang Anda harus miliki untuk mulai menggunakan Windows Azure:

1. Account Outlook (live, hotmail, etc.)

2. Account Azure

3. Account Dreamspark

4. Template WEB yang sudah siap upload (jika Ada).

5. Aplikasi untuk mengakses FTP (bisa juga menggunakan Expoler di Windows)

Jika sudah membuat atau memiliki akun Microsoft dan mendaftarkan di Azure, kamu harus mempunyai akun Dreamspark untuk mendapatkan akses gratis di Azure : Kunjungi situs DreamSpark, Lalu klik tab student, dan create Account. Setelah pendaftaran Anda berhasil, Anda harus memverifikasi data Anda sebagai pelajar dengan memasukkan alamat email sekolah Anda. Selain itu aktivasi juga dapat dilakukan dengan kartu ISIC atau memasukkan kode verifikasi DreamSpark (jika Ada).

Selanjutnya Anda perlu menghubungkan akun DreamSpark Anda dengan Azure, yaitu dengan mengunjungi katalog yang disediakan seperti di bawah ini.

Jika akun sudah diverifikasi (lihat gambar di bawah ini), silakan menuju ke portal Azure

Langkah membuat Subscibtion dan Azure WEB+Apps Services

Setelah menuju ke portal Azure buatlah project baru yaitu dengan klik "New\Web + Mobile\Web App" Lalu isikan kolom yang telah disediakan.

*App Name, Adalah nama url/domain Anda yang nantinya berakhiran .azurewebsites.net

*Subscription, Adalah layanan sebagai Subscription Anda di DreamSpark

*App Service Plan, adalah lokasi dari service Anda. Lalu klik Create. Dan tunggu hingga proses selesai.

Setelah semua proses selesai, diwajibkan mengatur kredensial Anda untuk dapat mengakses data melalui FTP.

Klik Deployment Credentials pada menu PUBLISHING, lalu isi kolom yang disediakan dan klik Save.

*FTP/deployment user name, digunakan sebagai ID Anda ketika login ke FTP

*Password, begitu juga dengan Password. Adalah kata sandi Anda ketika mengakses FTP.


· Langkah mengakses dan meng-upload file Web melalui FTP

Pada tutorial kali ini saya menggunakan FileZilla sebagai alat untuk mengakses FTP pada Azure. Yang perlu Anda lakukan adalah, memasukkan akun FTP Anda ke FileZilla dengan cara.

1. Buka FileZilla

2. File->Site Manager

3. Isikan Host dengan url Hostname yang berada pada informasi aplikasi Anda

4. Pilih Normal Logon Type

5. Lalu masukkan password FTP yang Anda buat sebelumnya. Klik Connect dan tunggu hingga FileZilla berhasil terhubung

6. Lalu upload file WEB yang sudah Anda buat sebelumnya di public directory web Anda ke /site/wwwroot

Itulah tutorial yang saya berikan mengenai "Tutorial menggunakan Azure WEB+Apps Services. Cara di atas adalah cara yang saya lakukan ketika saya meng-upload WEB ke Azure Services menggunakan FileZilla. Untuk meng-Upload File WEB Anda juga dapat men-Drag File tersebut langsung ke Public Directory. Dalam waktu dekat akan saya buat postingan Tutorial menggunakan Database dari Azure. Selamat mencoba!

Keamanan di Dunia

Berbasis Cloud

Pada kesempatan kali ini saya akan menjelaskan secara singkat apa yang saya dapatkan setelah menyelesaikan Course di MVA (Microsoft Virtual Academy) mengenai Keamanan di Dunia Berbasis Cloud.

Berbicara soal keamanan dunia Cloud, pastilah perlu dilakukan untuk menjaga sebuah data, aplikasi, dan jaringan agar tetap aman apalagi data tersebut penting bahkan aset dari suatu perusahaan/organisasi. Pada saat suatu organisasi menggunakan layanan Cloud sebagai penyimpanan data dan lainnya (termasuk data yang sensitif), organisasi tersebut seolah menaruh kepercayaan dan bergantung pada suatu penyedia layanan Cloud. Dunia Cloud juga tidak terlepas dari hal yang dapat merugikan seperti tindak kejahatan seperti pencurian, peretasan, kebocoran data, dan lainnya. Tidak terlepas dari kemungkinan, kejadian tersebut menjadi tanggung jawab Pengguna bahkan penyedia layanan Cloud. Lalu apa saja yang menjadi tanggung jawab pengguna dan penyedia layanan cloud?

Sebelum saya melanjutkan ada baiknya kita mengetahui apa saja layanan yang terdapat pada Cloud, karena Tanggung jawab dan kontrol untuk keamanan data, aplikasi, dan jaringan bervariasi menurut jenis layanan di antaranya, SaaS, PaaS, dan IaaS.

SaaS (Software as a Service)*
Software as a Service adalah layanan komputasi awan dimana kita bisa langsung menggunakan aplikasi yang telah disediakan. Penyedia layanan mengelola infrastruktur dan platform yang menjalankan aplikasi tersebut. Contoh layanan aplikasi email yaitu gmail, yahoo dan outlook sedangkan contoh aplikasi media sosial adalah twitter, facebook dan google+. Keuntungan dari layanan ini adalah pengguna tidak perlu membeli lisensi untuk mengakses aplikasi tersebut. Pengguna hanya membutuhkan perangkat klien komputasi awan yang terhubung ke internet.

PaaS (Platform as a Service)*
Platform as a Service adalah layanan yang menyediakan computing platform. Biasanya sudah terdapat sistem operasi, database, web server dan framework aplikasi agar dapat menjalankan aplikasi yang telah dibuat. Perusahaan yang menyediakan layanan tersebutlah yang bertanggung jawab dalam pemeliharaan computing platform ini. Keuntungan layanan PaaS ini bagi pengembang adalah mereka bisa fokus pada aplikasi yang mereka buat tanpa memikirkan tentang pemeliharaan dari computing platform.

IaaS (Infrastucture as a Service)*
Infrastructure as a Service adalah layanan komputasi awan yang menyediakan infrastruktur IT berupa CPU, RAM, storage, bandwith dan konfigurasi lain. Komponen-komponen tersebut digunakan untuk membangun komputer virtual. Komputer virtual dapat diinstal sistem operasi dan aplikasi sesuai kebutuhan. Keuntungan layanan IaaS ini adalah tidak perlu membeli komputer fisik sehingga lebih menghemat biaya. Konfigurasi komputer virtual juga bisa diubah sesuai kebutuhan. Misalkan saat storage hampir penuh, storage bisa ditambah dengan segera.
